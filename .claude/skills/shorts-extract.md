# 쇼츠 추출 스킬

보급형피아노쌤 채널의 완성된 영상에서 유튜브 쇼츠 구간을 추출하고, Premiere Pro MCP로 서브시퀀스를 자동 생성하는 워크플로우를 실행한다.

## 실행 순서

### 1단계 — SRT 파일 확인
사용자에게 묻는다: "SRT(또는 타임스탬프 포함 텍스트) 파일 경로를 알려주세요."
파일을 읽고 전체 내용을 파악한다.

### 2단계 — 구간 분석 및 목록 제시

**포함 기준 (모두 포함, 절대 임의 제외 금지):**
- 교육/정보성 내용
- 비유·스토리·강사 경험담
- 채널 홍보, 레슨 모집 → 반드시 포함

**제외 기준:**
- 매 영상 반복되는 공식 인사 ("안녕하세요 여러분, 보급형피아노쌤입니다") — 시작점에서 제외
- 5초 이하 단순 인사/아웃트로
- 앞뒤 맥락 없이 이해 불가능한 토막

**연주 전용 구간 처리:**
- 설명 없이 연주만 있는 구간도 포함한다
- 앞 구간의 설명과 이어지는 예시 연주라면 설명 구간과 묶어서 하나의 쇼츠로 구성
- 독립된 연주 구간(정리 파트 재연주, 솔로 연주 등)도 단독 쇼츠로 포함 가능
- 단, 5초 이하이거나 완전히 무음인 구간은 제외

**길이 기준:** 15~60초 (61초 허용, 초과 시 분할 검토)

**제목 작성 규칙 (핵심):**
- 반드시 해당 구간 SRT의 실제 발화를 직접 인용해서 제목 작성
- 위치(도입부·중반 등)나 흐름으로 추측해서 제목 짓는 것 금지
- 근거 발화를 표에 반드시 병기

아래 형식으로 전체 제시:

| # | 제목 | 구간 | 길이 | 근거 발화 |
|---|------|------|------|-----------|
| 1 | 제목 | 00:00:00 ~ 00:00:00 | 00초 | "실제 SRT 문장" |

### 3단계 — 사용자 확인
"빼거나 수정할 구간 있으면 말씀해 주세요. 없으면 바로 생성합니다."

### 4단계 — MCP로 서브시퀀스 생성

**사전 확인:**
- MCP Bridge가 실행 중인지 사용자에게 확인 (Premiere Pro → Window → Extensions → MCP Bridge → Start Bridge)
- `get_project_info`로 메인 시퀀스 ID 확인

**겹침 금지 원칙 (필수):**
- 반드시 이전 서브시퀀스의 out = 다음 서브시퀀스의 in
- 어떤 구간도 다른 구간과 겹쳐서는 안 됨
- 생성 전 체크: 모든 구간이 연속적이고 겹침 없는지 확인

**각 서브시퀀스 생성 순서 (3단계, 반드시 순서대로):**
1. `set_sequence_in_out_points` — 메인 시퀀스에 in/out 설정
2. `create_subsequence` — 서브시퀀스 생성 (이름: Sub_01, Sub_02, ...)
3. `add_text_overlay` — MOGRT 배치
   - mogrtPath: `C:/Users/PC/AppData/Roaming/Adobe/Common/Motion Graphics Templates/보피쌤 쇼츠용 모션그래픽.mogrt`
   - trackIndex: 1, startTime: 0, duration: 서브시퀀스 길이(초)
   - ※ 텍스트 자동 주입 불가 (appHasMOGRTAPI: false) — MOGRT 배치만 되고 텍스트는 사용자가 수동 입력

### 5단계 — 완료 후 프로젝트 저장
모든 서브시퀀스 생성 완료 후 `save_project` 실행.

사용자에게 수동 입력할 텍스트 목록 제공:
| # | 서브시퀀스 | 입력할 텍스트 |
|---|-----------|-------------|
| 1 | Sub_01 | 제목 |

## 알려진 한계
- MOGRT 한글 텍스트 자동 주입 불가 (`appHasMOGRTAPI: false`)
- 각 서브시퀀스는 3번의 순차 MCP 호출 필요 → 병렬 처리 불가 → 25개면 75번 순차 호출
