# 쇼츠 추출 규칙

> 완성된 영상에서 쇼츠 구간을 최대한 많이 뽑기 위한 가이드.
> 새 세션에서 이 파일을 참조하면 바로 이어서 작업 가능.

---

## 핵심 원칙

**"이 구간만 봐도 하나의 완결된 내용인가?"** — 이게 유일한 기준.

- 길이: **15~60초** (61초 정도는 허용. 60초 초과 시 분할 검토)
- 목표: 독립적으로 쓸 수 있는 구간을 **최대한 많이** 추출

---

## 포함 기준

| 유형 | 예시 | 비고 |
|------|------|------|
| 교육/정보성 | 개념 설명, 팁, 연습법 | 기본 |
| 비유/스토리 | "메트로놈 = 제대로 걷기" | 공감 유도형 |
| 강사 개인 경험 | 레슨 생활, 실패담 | 친근감 |
| 채널 홍보/레슨 모집 | "줌레슨 신청하세요" | **채널 홍보 쇼츠로 활용 가능 → 반드시 포함** |

---

## 제외 기준

- 5초 이하 단순 인사/아웃트로 (내용 없음)
- 앞뒤 맥락 없이는 이해 불가능한 중간 토막
- 연주만 있고 설명이 없는 구간 (영상에서 판단)

---

## 클로드가 분석 시 하지 말아야 할 것

| 하면 안 되는 행동 | 이유 |
|------------------|------|
| "광고성이다" → 제외 | 채널 홍보 쇼츠로 활용 가능 |
| "정보성 콘텐츠만" 기준으로 판단 | 기준이 너무 좁음 |
| "이건 쇼츠에 안 맞겠다"고 임의 판단 | 사용자가 판단할 일 |
| 구간 목록 일부 누락 | 전체 제시 후 사용자가 선택 |

→ **클로드는 가능한 구간을 전부 나열하고, 선택은 사용자가 한다.**

## 쇼츠 분석 워크플로우

### Step 1 — 파일 읽기
SRT 또는 타임스탬프 포함 텍스트 파일 읽기.
SRT 없으면 Whisper로 먼저 생성:
```bash
whisper "영상.mp4" --language Korean --model medium --output_format srt --output_dir "자막/"
```

### Step 2 — 전체 분석
영상 전체 내용 파악 → 독립 가능한 **모든** 구간 추출.

### Step 3 — 목록 제시
아래 형식으로 전체 제시:

| # | 제목 | 구간 | 길이 |
|---|------|------|------|
| 1 | 제목 | 00:00:00 ~ 00:00:00 | 00초 |

### Step 4 — 사용자 확인
빼거나 조정할 구간 확인 후 확정.

### Step 5 — MCP로 서브시퀀스 생성
```
툴: create_subsequence
시퀀스 ID: get_project_info로 확인 (매 프로젝트마다 다름)
이름 규칙: Sub_01, Sub_02, ...
```

### Step 6 — 텍스트 오버레이 (선택)
`add_text_overlay`로 제목 삽입 시도.
안 되면 컷편집만으로 완료.
