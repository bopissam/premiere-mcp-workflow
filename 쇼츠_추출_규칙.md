# 쇼츠 추출 규칙

> 완성된 영상에서 쇼츠 구간을 최대한 많이 뽑기 위한 가이드.
> 새 세션에서 이 파일을 참조하면 바로 이어서 작업 가능.

---

## 핵심 원칙

**"이 구간만 봐도 하나의 완결된 내용인가?"** — 이게 유일한 기준.

- 길이: **15~60초** (61초 정도는 허용. 60초 초과 시 분할 검토)
- 목표: 독립적으로 쓸 수 있는 구간을 **최대한 많이** 추출

---

## 포함 기준

| 유형 | 예시 | 비고 |
|------|------|------|
| 교육/정보성 | 개념 설명, 팁, 연습법 | 기본 |
| 비유/스토리 | "메트로놈 = 제대로 걷기" | 공감 유도형 |
| 강사 개인 경험 | 레슨 생활, 실패담 | 친근감 |
| 채널 홍보/레슨 모집 | "줌레슨 신청하세요" | **채널 홍보 쇼츠로 활용 가능 → 반드시 포함** |

---

## 제외 기준

- 5초 이하 단순 인사/아웃트로 (내용 없음)
- 매 영상 반복되는 공식 인사 ("안녕하세요 여러분, 보급형피아노쌤입니다") — 시작점에서 제외
- 앞뒤 맥락 없이는 이해 불가능한 중간 토막
- 연주만 있고 설명이 없는 구간

---

## 클로드가 분석 시 하지 말아야 할 것

| 하면 안 되는 행동 | 이유 |
|------------------|------|
| "광고성이다" → 제외 | 채널 홍보 쇼츠로 활용 가능 |
| "정보성 콘텐츠만" 기준으로 판단 | 기준이 너무 좁음 |
| "이건 쇼츠에 안 맞겠다"고 임의 판단 | 사용자가 판단할 일 |
| 구간 목록 일부 누락 | 전체 제시 후 사용자가 선택 |
| SRT 확인 없이 제목을 추측으로 작성 | 실제 발화와 다른 제목이 나옴 (오늘 Sub_02 사태 참고) |
| 서브시퀀스 구간을 겹치게 설정 | 같은 내용이 두 쇼츠에 중복 포함됨 |

→ **클로드는 가능한 구간을 전부 나열하고, 선택은 사용자가 한다.**

## 쇼츠 분석 워크플로우

### Step 1 — 파일 읽기
SRT 또는 타임스탬프 포함 텍스트 파일 읽기.
SRT 없으면 Whisper로 먼저 생성:
```bash
whisper "영상.mp4" --language Korean --model medium --output_format srt --output_dir "자막/"
```

### Step 2 — 전체 분석
영상 전체 내용 파악 → 독립 가능한 **모든** 구간 추출.

### Step 3 — 목록 제시 (제목은 반드시 SRT 실제 발화 기반)

**제목 작성 2원칙:**
1. 해당 구간 SRT에서 실제로 나온 발화/문장을 직접 인용해서 만들 것
2. 내용을 추측하거나 "도입부니까 개요겠지" 같은 위치 기반 추론 금지

아래 형식으로 전체 제시:

| # | 제목 | 구간 | 길이 | 근거 발화 |
|---|------|------|------|-----------|
| 1 | 제목 | 00:00:00 ~ 00:00:00 | 00초 | "실제 SRT 문장" |

### Step 4 — 사용자 확인
빼거나 조정할 구간 확인 후 확정.
제목이 어색하면 이 단계에서 수정 (근거 발화가 있으면 확인이 빠름).

### Step 5 — 서브시퀀스 생성 (겹침 없이, 순서대로)

**겹침 금지 원칙:**
- 서브시퀀스는 반드시 **이전 서브시퀀스가 끝나는 지점부터** 시작
- 예시: Sub_01이 0:17~1:41이면, Sub_02는 반드시 1:41~부터 시작
- 같은 내용이 두 쇼츠에 들어가는 일 없도록 — 겹치면 콘텐츠 중복이 됨

**생성 전 체크리스트:**
```
[ ] 각 구간의 in/out 타임코드가 확정됐는가?
[ ] 이전 구간의 out = 다음 구간의 in 인가?
[ ] 어떤 구간도 다른 구간과 겹치지 않는가?
```

```
툴: create_subsequence
시퀀스 ID: get_project_info로 확인 (매 프로젝트마다 다름)
이름 규칙: Sub_01, Sub_02, ...
순서: set_sequence_in_out_points → create_subsequence → add_text_overlay (3단계 반복)
```

### Step 6 — MOGRT 텍스트 오버레이 (자동 배치, 텍스트는 수동)

`add_text_overlay`로 MOGRT 배치 → 텍스트 내용은 사용자가 수동 입력.

MOGRT 경로: `C:/Users/PC/AppData/Roaming/Adobe/Common/Motion Graphics Templates/보피쌤 쇼츠용 모션그래픽.mogrt`

trackIndex: 1 (V2), startTime: 0, duration: 서브시퀀스 길이(초)
